طوّر هذا الموقع الحالي مباشرةً ولا تبدأ من الصفر.

أريد تحويله من نسخة عادية إلى موقع قوي بمستوى عالمي، مع تحسينات حقيقية في:
- التصميم
- الأداء
- تجربة المستخدم
- الأمان
- الموثوقية
- قابلية التوسع
- المزايا الحديثة

نفّذ العمل داخل المشروع الحالي فقط.

المطلوب بالتسلسل:

1) تحليل شامل للموقع الحالي
- افحص الهيكل الحالي للموقع
- حدّد نقاط الضعف في UI/UX
- حدّد نقاط الضعف في الأداء
- حدّد نقاط الضعف في الأمان
- حدّد الصفحات أو الأجزاء التي تبدو بدائية أو غير مكتملة
- لا تكتب خطة طويلة فقط، بل انتقل بعدها مباشرة للتنفيذ

2) تحسين الواجهة بمستوى عالمي
- حسّن التصميم الحالي بدون هدم المشروع
- اجعل الواجهة احترافية ونظيفة وحديثة
- حسّن typography, spacing, colors, buttons, cards, forms
- حسّن responsive design للموبايل والتابلت والديسكتوب
- حسّن RTL إذا كان الموقع عربيًا
- أضف empty states / loading states / error states احترافية
- اجعل التنقل واضحًا وسريعًا

3) إضافة مزايا قوية عالمية عند الحاجة
ابحث عمّا يناسب نوع الموقع وأضف أهم الإضافات المفيدة، مثل:
- PWA support
- offline support حيث يناسب
- push notifications حيث يناسب
- search متقدم
- filters ذكية
- dashboard metrics
- audit logs
- activity timeline
- export PDF / Excel / CSV
- dark mode
- role-based access
- analytics
- multi-language readiness
- onboarding walkthrough
- feedback/report issue widget
- file upload improvements
- image optimization
- quick actions
- keyboard shortcuts في لوحات التحكم
- autosave في النماذج
- breadcrumbs
- skeleton loaders

اختر فقط ما يناسب هذا المشروع، ولا تضف ميزات عشوائية.

4) البحث عن إضافات ومكتبات مناسبة
- ابحث عن أفضل المكتبات أو الإضافات المناسبة للمشروع الحالي
- أضف فقط المكتبات المستقرة والمفيدة
- تجنب الإضافات الثقيلة أو غير الضرورية
- إن أضفت مكتبة، فاذكر لماذا أضفتها وما فائدتها

5) تحسين الأداء
- حسّن سرعة التحميل
- قلل الحجم غير الضروري
- lazy loading
- code splitting إذا كان مناسبًا
- image optimization
- caching strategy
- تحسين Core Web Vitals
- إزالة أي أكواد أو dependencies غير مستخدمة

6) تحسين الأمان
- راجع auth / forms / API calls
- أضف حماية أساسية قوية
- security headers
- validation
- sanitization
- error handling
- حماية صفحات الإدارة
- لا تترك أي secrets مكشوفة في الواجهة
- طبّق أفضل الممارسات المستندة إلى OWASP بقدر ما يناسب المشروع

7) تحسين الجودة العامة
- أصلح أي روابط مكسورة
- أصلح أي شاشات ناقصة
- أصلح أي بيانات mock ظاهرة للمستخدم إن أمكن
- حسّن الرسائل العربية والنصوص
- وحّد أسلوب الواجهة كاملًا
- حسّن حالات النجاح والفشل والتنبيهات

8) الاختبار النهائي
اختبر:
- الموبايل
- الديسكتوب
- التنقل
- النماذج
- الأداء
- الأجزاء الجديدة التي أضفتها

9) التسليم النهائي
في النهاية أعطني فقط:
- الملفات التي عدلتها
- الإضافات أو المكتبات التي أضفتها
- أهم التحسينات الفعلية
- ما الذي اختبرته
- ما الذي بقي كمرحلة تالية
